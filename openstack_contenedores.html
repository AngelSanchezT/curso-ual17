Hypervisor LXC

  • Usar LCX como hipervisor

LXC:utiliza técnicas de virtualización a nivel de sistema operativo, de modo que sólo hay un núcleo, en nuestro caso el Linux que se ejecuta en el equipo anfitrión, que mediante diferentes funciones (namespaces, cgroups, ...) muestra vistas parciales a algunos procesos para que crean que se ejecutan en un sistema aislado.

  • Se gestionan sobre libvirt
  • El proyecto de usar Docker como supervisor ha desaparecido en OpenSatck

Ventajas

  • Podemos crear muchas más contenedores que MV
  • La creación de los contenedores es mucho más rápida
  • Podemos tener nodos de computo con KVM y otros con LXC (zonas distintas)
